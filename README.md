# DUNGEON QUEST

**ローグライク × JRPG** のブラウザゲーム。純粋な HTML / CSS / JS で動作し、サーバー不要で `index.html` をブラウザで開くだけでプレイできます。

---

## ゲームの概要

7つの階層からなるダンジョンを攻略し、最深部に潜む龍神を討伐することが目的です。
限られた回復手段の中でリソースを管理しながら進む、高難度なローグライク体験を目指しています。

---

## ゲームシステム

### 階層構成

| 階層 | 内容 |
|---|---|
| 第1〜6階層 | 戦闘・イベント・階段の3択からランダムで進む |
| 第7階層 | ボス戦のみ（龍神ヴァルムドラグ） |

- 各階層の**1回目は必ず戦闘**からスタート
- **階段**で次の階層へ進める（同じ階層に連続して階段は出ない）
- 階層移動のタイミングで **HP・MP が全回復**
- 階層移動後に **ショップ** でレリック・アイテムを購入できる

### 戦闘システム（コマンドバトル）

| コマンド | 説明 |
|---|---|
| 攻撃 | 通常物理攻撃 |
| 防御 | そのターンのダメージを大幅軽減 |
| スキル | MP消費で強力な技を使用 |
| アイテム | 所持しているアイテムを使用 |

- ターン制（プレイヤー先攻）
- 同じ階層で戦闘を重ねるほど敵が強くなる（HP +12% / ATK +6% / 戦闘ごと）
- ボスは専用スキル（ドラゴンブレス・尻尾攻撃・咆哮）を使用する

### ショップ

階層移動のたびにラインナップが入れ替わる。

| カテゴリ | 内容 |
|---|---|
| レリック | ステータス増加アイテム（複数所持可・上限なし） |
| アイテム | 消耗品（ポーション・爆弾・毒の小瓶など） |

- 入店ごとに**レリック5個 + アイテム3個**をランダム抽選
- レアリティ（tier）が高いほど出にくい

### レリック

複数個所持でき、所持しているだけで効果が発動する。

| レリック | 効果 | 価格 |
|---|---|---|
| 鉄の剣 | ATK+8 | 40G |
| 鋼の剣 | ATK+18 | 90G |
| 炎の剣 | ATK+28 | 150G |
| 革の鎧 | DEF+6 | 35G |
| 鉄の鎧 | DEF+18 | 95G |
| 生命の指輪 | MaxHP+35 | 45G |
| 魔法のローブ | DEF+8 / MaxMP+20 | 80G |
| 竜鱗の欠片 | ATK+12 / DEF+12 | 120G |

### イベント

各階層にランダムで発生。選択肢によって効果が変わる。

| 種類（例） | 内容 |
|---|---|
| 宝箱 | ゴールド獲得 or トラップ or ミミック |
| バフ系 | 攻撃力・防御力の一時強化 |
| 運命の輝き | ステータス強化 or 弱体化（選択） |
| 血の盟約 | HP削りと引き換えにゴールド2倍 |
| 黄金の盾 | ダメージをゴールドで肩代わり |
| 富の力 | 所持ゴールド量に応じて攻撃力UP（動的変動） |
| 落とし穴 | HP/MP回復・ショップなしで強制的に次階層へ |
| 奇妙な商人 | アイテムを交換 |
| 魔力解放（4階） | MP全消費で攻撃・防御3倍（この階層中） |
| 覇者の瞳（4階） | 次の2戦闘クリティカル率30% |
| スリ（4階） | ゴールドの50%を失う |
| 血の鎧（5階） | 攻撃2倍・毎ターン10ダメージ（この階層中） |
| 攻の契約（5階） | 攻撃×1.5・防御×0.7（この階層中） |
| 守の契約（5階） | 防御×2・攻撃×0.7（この階層中） |
| 強敵の試練（6階） | 次の敵が強化、勝利でステータス永続1.3倍 |
| 瀕死の覚醒（6階） | HPが低いほど攻撃力最大2倍（この階層中） |
| スキルの賭け（6階） | 50%でスキル効果2倍 or スキル使用不可（この階層中） |

## スコア計算

```
スコア = (到達階層 - 1) × 600 + 総獲得ゴールド × 2 + 総戦闘回数 × 60
ボスクリア時 +5000 ボーナス
```

## 技術スタック

- **言語**: HTML / CSS / JavaScript（ライブラリなし）
- **描画**: Canvas 2D API（ドット絵スプライトは手描き）
- **対応環境**: モダンブラウザ（Chrome / Firefox / Safari / Edge）

## ファイル詳細

### `index.html`
ゲーム全体のHTML骨格。タイトル・バトル・ショップ・イベント・階段・ゲームオーバー・クリアの各シーンを `<div id="scene-xxx">` で管理し、`hidden` クラスで表示切り替えを行う。

### `css/style.css`
全画面のスタイル定義。ダークなピクセルアート風UIをCSSのみで実現。HPバー・コマンドボタン・ショップタブ・階段確認画面などのレイアウトを含む。

---

### `js/game.js` ― ゲームコア（共通）

| 要素 | 説明 |
|---|---|
| `GS` オブジェクト | 全ファイルから参照する共有ゲームステート（HP・MP・ゴールド・レリックなど） |
| `resetGame()` | ゲーム開始時の初期化 |
| `showScene(name)` | シーン切り替え |
| `initFloorSelect()` | 戦闘・イベント・階段をランダム選択して遷移 |
| `descendFloor(skipShop)` | 階層移動・HP/MP全回復・ショップへ（落とし穴時はスキップ） |
| `gotoGameOver()` / `gotoVictory()` | 終了画面の表示 |
| `pct()` / `makeMenuBtn()` | 全ファイルから使うユーティリティ関数 |

### `js/battle.js` ― バトルロジック（担当B）

| 要素 | 説明 |
|---|---|
| `spawnEnemy()` | 階層・戦闘回数に応じてスケールした敵を生成 |
| `initBattle()` | バトル画面の初期化 |
| `doAttack()` / `doDefend()` / `doSkill()` / `doItem()` | プレイヤーの各行動 |
| `doEnemyTurn()` | 敵のターン処理（ボスは専用スキルあり） |
| `checkWin()` / `checkLose()` | 勝敗判定・報酬付与 |
| `calcPhysDmg()` / `calcMagicDmg()` | ダメージ計算式 |
| `startBattleAnim()` | requestAnimationFrame によるアニメーションループ |
| `drawPlayer()` / `drawEnemy()` など | Canvas手描きスプライト（プレイヤー＋敵20種） |

### `js/shop.js` ― ショップ処理（担当A）

| 要素 | 説明 |
|---|---|
| `initShop()` / `renderShop()` | ショップ画面の表示・タブ切り替え |
| `generateLineup()` | 入店ごとにレリック5個＋アイテム3個をランダム抽選 |
| `isOwned()` | 購入済みレリックの判定 |
| `doBuy()` | 購入処理（レリック・アイテムで処理が異なる） |

### `js/events.js` ― イベント処理（担当C）

| 要素 | 説明 |
|---|---|
| `FLOOR_EVENTS` | 階層別イベントプール（1〜6階すべて実装済み、各5〜6種） |
| `initEvent()` | 階層に応じたイベントをランダム選択・選択肢を動的に描画 |
| `_resolveAfterEvent()` | 選択後の遷移処理（落とし穴・ミミック等の特殊フラグを処理） |

---

### `data/enemies.js` ― 敵データ（担当B）

| 要素 | 説明 |
|---|---|
| `ENEMIES` | 19種の敵データ。HP・攻撃・防御・報酬ゴールド・スプライト名などを定義 |
| `FLOOR_ENEMIES` | 階層ごとの出現敵リスト（1〜6階各3種・7階ボス固定） |

### `data/skills.js` ― スキルデータ（担当C）

| 要素 | 説明 |
|---|---|
| `SKILLS` | 6種のスキル定義。MPコスト・タイプ（物理/魔法/回復/バフ）・威力など |

### `data/items.js` ― アイテムデータ（担当B）

6種の消耗品定義（ポーション・ハイポーション・エーテル・爆弾・毒の小瓶・蘇生草）。`type` フィールドでバトル中の使用効果が変わる。

### `data/relics.js` ― レリックデータ（担当A）

8種のレリック定義。ATK・DEF・MaxHP・MaxMPのボーナスを持ち、複数所持可能。特殊効果（パッシブ）は今後追加予定。
