## ファイル構成

```
rogulike/
├── index.html           # 全シーンのHTML構造・スクリプト読み込み
├── css/
│   └── style.css        # ダークピクセルアートUI・全画面レイアウト
├── data/
│   ├── enemies.js       # 敵データ・階層別出現テーブル（19種）
│   ├── skills.js        # スキル定義（15種）
│   ├── items.js         # アイテムデータ（消耗品6種）
│   └── relics.js        # レリックデータ（79種）
└── js/
    ├── game.js          # ゲームコア・シーン管理・共有ステート
    ├── battle.js        # バトルロジック・Canvasスプライト描画（20種）
    ├── shop.js          # ショップ処理・ランダムラインナップ
    └── events.js        # 階層別イベントデータ・処理
```

## ファイル詳細

### `js/game.js` ― ゲームコア（共通）

| 要素 | 説明 |
|---|---|
| `GS` オブジェクト | 全ファイルから参照する共有ゲームステート（HP・MP・ゴールド・レリックなど） |
| `resetGame()` | ゲーム開始時の初期化 |
| `showScene(name)` | シーン切り替え |
| `initFloorSelect()` | 戦闘・イベント・階段をランダム選択して遷移 |
| `descendFloor(skipShop)` | 階層移動・HP/MP全回復・ショップへ（落とし穴時はスキップ） |
| `gotoGameOver()` / `gotoVictory()` | 終了画面の表示 |
| `pct()` / `makeMenuBtn()` | 全ファイルから使うユーティリティ関数 |

### `js/battle.js` ― バトルロジック（担当B）

| 要素 | 説明 |
|---|---|
| `spawnEnemy()` | 階層・戦闘回数に応じてスケールした敵を生成 |
| `initBattle()` | バトル画面の初期化 |
| `doAttack()` / `doDefend()` / `doSkill()` / `doItem()` | プレイヤーの各行動 |
| `doEnemyTurn()` | 敵のターン処理（ボスは専用スキルあり） |
| `checkWin()` / `checkLose()` | 勝敗判定・報酬付与 |
| `calcPhysDmg()` / `calcMagicDmg()` | ダメージ計算式 |
| `startBattleAnim()` | requestAnimationFrame によるアニメーションループ |
| `drawPlayer()` / `drawEnemy()` など | Canvas手描きスプライト（プレイヤー＋敵20種） |

### `js/shop.js` ― ショップ処理（担当A）

| 要素 | 説明 |
|---|---|
| `initShop()` / `renderShop()` | ショップ画面の表示・タブ切り替え |
| `generateLineup()` | 入店ごとにレリック＋アイテムをランダム抽選 |
| `isOwned()` | 購入済みレリックの判定 |
| `doBuy()` | 購入処理 |

### `js/events.js` ― イベント処理（担当C）

| 要素 | 説明 |
|---|---|
| `FLOOR_EVENTS` | 階層別イベントプール（1〜6階すべて実装済み、各5〜6種） |
| `initEvent()` | 階層に応じたイベントをランダム選択・選択肢を動的に描画 |
| `_resolveAfterEvent()` | 選択後の遷移処理（落とし穴・ミミック等の特殊フラグを処理） |

|---|---|

## スコア計算

```
スコア = (到達階層 - 1) × 600 + 総獲得ゴールド × 2 + 総戦闘回数 × 60
ボスクリア時 +5000 ボーナス
```
