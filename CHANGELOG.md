# CHANGELOG

## [v0.11.0] - 2026-04-08

### 修正（担当C）
- 1層の「戦士の覚醒」「守護の気配」バフのカウントダウンタイミングを修正
  - 従来：戦闘「開始時」にカウントダウン → 2戦目開始と同時にバフが消えるバグ
  - 修正後：戦闘「終了時」（`checkWin()`）にカウントダウン → 2戦闘の間フル持続
  - 同様の修正を `mpFreeRemain`（MP無消費）にも適用
- 3層「富の力」のゴールド量→攻撃力変換を動的計算に変更
  - 従来：イベント発生時点のゴールド量で倍率を固定
  - 修正後：攻撃のたびに現在のゴールド量から倍率を動的に計算（所持Gが変わると攻撃力も即反映）

### 追加（担当B・C）
- 4〜6層のイベントを本実装（各5種類、合計15イベント追加）

### 追加（担当B）
- 新規プレイヤー状態フィールドを追加（game.js / battle.js）
  - `critBuffRemain`：クリティカル率バフの残り戦闘数管理
  - `damageTakenMult`：受けるダメージ倍率（初期値1）
  - `turnDmg`：毎ターン受ける固定ダメージ（血の鎧用）
  - `skillPowerMult`：スキル効果倍率（6層スキル賭け用）
  - `skillDisabled`：スキル全使用不可フラグ（6層スキル賭け用）
  - `mpCostMult`：MP消費倍率（6層トラップ用）
  - `hpLowAtkActive`：HP低下に応じた攻撃力上昇フラグ（6層用）
  - `challengeBattle`：挑戦戦闘フラグ（6層試練用）
- `applyDmgToPlayer()` に `damageTakenMult` 適用を追加
- `doEnemyTurn()` の開始時に `turnDmg` による毎ターンダメージを追加
- `openSkillMenu()` に `skillDisabled` チェックと `mpCostMult` 表示を追加
- `doSkill()` に `mpCostMult` と `skillPowerMult` の適用を追加
- `spawnEnemy()` に `challengeBattle` 時の敵強化（攻防×1.5）を追加
- `checkWin()` に挑戦戦闘勝利時のステータス×1.3ボーナスを追加

## [v0.7.0] - 2026-04-05

### 追加（担当A）
- レリックシステムの導入
  - 武器・防具を廃止し「レリック」に統合（複数所持可、上限なし）
  - レリック8種：鉄の剣 / 鋼の剣 / 炎の剣 / 革の鎧 / 鉄の鎧 / 生命の指輪 / 魔法のローブ / 竜鱗の欠片
  - tier1〜3の重み付けによりレア度を設定
- ショップシステムの刷新
  - タブを「レリック」「アイテム」の2つに統合（スキルタブ廃止）
  - 入店ごとにレリック5個＋アイテム3個をランダム抽選（ラインナップが毎回変わる）
  - 価格を全体的に引き下げ、購入しやすく調整

### 追加（担当C）
- イベントシステムの本実装
  - 階層別イベントプール（各階層5〜6種類）
  - 各イベントに選択肢を設置、選択結果をその場で表示
  - 実装済みイベント一覧：
    - 1階層：宝箱 / 守護の気配（防御バフ） / 運命の輝き（ステータス強化 or 弱体化） / 戦士の覚醒（攻撃バフ） / 罠宝箱
    - 2階層：宝箱 / 血の盟約（HP削りでゴールド2倍） / 強敵の予感（敵強化+報酬2倍） / 魔力解放の石（MP消費なし） / 落とし穴宝箱 / 黄金の盾
    - 3階層：宝箱 / 覇者の契約（ゴールド消費で全ステ2倍） / 奇妙な商人（アイテム交換） / ミミック宝箱（戦闘突入） / 富の力（ゴールド量で攻撃倍率UP）
  - `oncePerFloor` フラグで1階層1回限りのイベントを制御
  - イベントプールが空のときは戦闘にフォールバック
